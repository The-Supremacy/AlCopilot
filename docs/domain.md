# AlCopilot Domain

## Purpose

This document is the thin domain index for AlCopilot.
Use it to understand the product domain in business terms and to navigate to module-specific domain guides.

---

## Product Domain

AlCopilot helps people discover, understand, and choose drinks.
The product domain centers on a curated drinks catalog, user-facing exploration, and AI-assisted suggestion flows grounded in understandable drink data.

Domain documents define business vocabulary, module ownership, and conceptual boundaries.
They do not define technical design or supported behavior in place of OpenSpec specifications.

---

## Domain Documentation Rules

- Keep domain docs business-facing and non-technical.
- Use domain docs for vocabulary, concepts, actors, and ownership boundaries.
- Use OpenSpec specs as the source of truth for observable supported behavior and acceptance criteria.
- Use architecture docs for technical structure and implementation direction.
